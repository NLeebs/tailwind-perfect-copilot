## Spacing Scale

Declared values (multiples of 4 only). Scale matches existing component usage:

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, micro separators |
| sm | 8px | p-2 / gap-2 | Inline chip padding (vertical) |
| md | 16px | p-4 / gap-4 | Default element spacing |
| lg | 24px | p-6 / gap-6 | Card padding, section padding |
| xl | 32px | p-8 / gap-8 | Nav padding, large card padding |
| 2xl | 48px | py-12 / gap-12 | Major section breaks |
| 3xl | 64px | py-16 / gap-16 | Page-level vertical rhythm |

Exceptions:
- CodeCallout chip horizontal padding: `px-3` (12px) at base, `3xl:px-4` (16px) at 1920px — deliberately tighter than the standard md token to keep the chip compact inline.
- CodeCallout chip vertical padding: `py-1` (4px) at base, `3xl:py-2` (8px) at 1920px.

## Typography

Font sizes used in this phase are confined to the CodeCallout chip. All other type is inherited from existing components. The chip must be legible at TV distance at the 3xl breakpoint.

| Role | Base Size | 3xl Size | Weight | Line Height | Font |
|------|-----------|----------|--------|-------------|------|
| Chip / callout text | 13px (`text-[13px]`) | 16px (`3xl:text-base`) | 400 (regular) | 1.5 | Geist Mono (`font-mono`) |
| Label / caps (inherited) | 12px (`text-xs`) | 14px (`3xl:text-sm`) | 600 (semibold) | 1 | Geist Sans |

Source: CssTimeline.tsx, SlideLayout.tsx, NavCard.tsx — scanned for all existing size/weight declarations.

Weights declared by Phase 1: regular (400) for chip text; semibold (600) for inherited label/caps chrome. The h1 bold (700) weight is pre-existing infrastructure, not introduced by this phase, and is excluded from this table.

### CodeCallout Chip Colors (locked, source: CONTEXT.md D-01)

| State | Background | Text | Border |
|-------|-----------|------|--------|

## CodeCallout Component Specification

This is the primary visual deliverable of Phase 1. All properties are locked by CONTEXT.md decisions D-01 through D-07.

### Anatomy

```
┌─────────────────────────────────────────────────────────┐
│  bg-sky-500 rounded-lg px-4 py-2                        │  ← chip content (wraps)
└─────────────────────────────────────────────────────────┘
```

The chip sits **below** the demo element. It is layout-agnostic — each slide page controls the containing flex/grid layout.

### API (locked, source: D-05, D-06)

```tsx
// Only prop — named `classes` to avoid confusion with element className
<CodeCallout classes="bg-sky-500 rounded-lg px-4 py-2" />
```

No `className`, `size`, `variant`, or other props. Appearance is fixed across all slides.

### Rendered HTML structure

```tsx
<p className="...chip-classes...">
  {classes}
</p>
```

Single `<p>` element. No wrapper div. Callers manage layout context.

### Class string (complete, static — no dynamic construction)

```
// Base (all viewports)
inline-block font-mono text-[13px] leading-relaxed
bg-slate-100 text-slate-700 border border-slate-200
rounded-lg px-3 py-1
whitespace-pre-wrap break-all

// Dark mode (class-based @custom-variant, NOT prefers-color-scheme)
dark:bg-slate-800 dark:text-slate-300 dark:border-slate-700

// 3xl escalation (1920px — TV readability required)
3xl:text-base 3xl:px-4 3xl:py-2 3xl:rounded-xl
```

`break-all` ensures long class strings (e.g., `hover:bg-sky-50 dark:hover:bg-sky-900/30 focus:ring-2`) wrap within the chip width without overflow. Per D-03: never truncate.

`whitespace-pre-wrap` preserves intentional spacing in class strings for readability.

### Single-source usage pattern (documented, not enforced — D-07)

```tsx
// In slide page.tsx (RSC):
const CARD_CLASSES = "bg-white rounded-2xl p-6 shadow-sm border border-slate-200";

<div className={CARD_CLASSES}>…</div>
<CodeCallout classes={CARD_CLASSES} />
```

This pattern is to be documented in CLAUDE.md. The component itself cannot enforce it.

### RSC status

`CodeCallout` is a React Server Component. No `"use client"` directive. No props requiring browser APIs or event handlers.

## Dark Mode Contract

Dark mode is class-based only. The `@custom-variant dark (&:where(.dark, .dark *))` in globals.css is the single source of truth. The `@media (prefers-color-scheme: dark)` block is removed in this phase (INFRA-03, D-10).

All dark mode variants in CodeCallout use the `dark:` prefix, which resolves through the custom variant to `.dark *`. No `prefers-color-scheme` is referenced anywhere in new code.

---

## TV Readability Contract (3xl: escalations required)

Per CLAUDE.md and QA-01: all text and spacing must include `3xl:` escalations for 1920px display.

CodeCallout chip escalations declared above:
- Text: `text-[13px]` → `3xl:text-base` (16px)
- Padding: `px-3 py-1` → `3xl:px-4 3xl:py-2`
- Border radius: `rounded-lg` → `3xl:rounded-xl`
