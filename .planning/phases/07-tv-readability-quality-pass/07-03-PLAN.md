    Step 3 — If the build fails, diagnose the error:

    a. TypeScript error ("Type error: ..."):
       - Read the file and line number cited in the error
       - Fix the type issue — common causes: a className string accidentally broken into two lines during editing, or a JSX attribute malformed
       - Re-run yarn build

    b. Tailwind purge warning or missing class in output:
       - The most likely cause is a class that was dynamically constructed (e.g., template literal) rather than a complete static string
       - Find the offending class using the error message
       - Convert to a complete static class string or a lookup-table map entry
       - Re-run yarn build

    c. Next.js build error ("Error: ..."):
       - Read the full error message
       - Common cause: a page.tsx that accidentally received "use client" during editing, or an async component that is not properly awaited
       - Fix and re-run yarn build

    Do NOT proceed to Step 4 until yarn build exits 0.

    ```
    yarn lint
    ```
