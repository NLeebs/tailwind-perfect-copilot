## Spacing Scale

Declared values — all multiples of 4, expressed as Tailwind utility classes:

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `mt-1` / `gap-1` | Icon gaps (not used this phase) |
| sm | 8px | `mt-2` | Paragraph-to-text tight spacing (overline → caption) |
| md | 16px | `mt-4` | Element-to-element within a column (button → callout, card → callout) |
| lg | 24px | `mt-6` | Grid column internal groupings |
| xl | 32px | `mt-8` | 3xl: escalated md spacing (`3xl:mt-8`) |
| 2xl | 64px | `mt-16` | Section spacers between slide sections |
| 3xl-section | 96px | `3xl:mt-24` | 3xl: escalated section spacers |

Grid gaps:
- Two-column demo grid: `gap-6 3xl:gap-12` (24px / 48px at 3xl)
- Three-column problem grid: `gap-6 3xl:gap-12` (24px / 48px at 3xl)

Section spacers between all four content sections: `mt-16 3xl:mt-24`

Internal element spacers within columns: `mt-4 3xl:mt-8`

Exceptions:
- Caption below overline: `mt-2 3xl:mt-4` (tighter — label/caption pair, not section break)
- Card padding: `p-6 3xl:p-10` (already baked into CARD_CLASSES const)

Source: CONTEXT.md D-01/D-03, RESEARCH.md §Pattern 4 (section spacers), RESEARCH.md §3xl/4xl Escalation Patterns — verified from `src/app/what-is-tailwind/page.tsx`.

## Color

This project uses slate (neutral) as dominant, with cyan as the single accent. No separate theme library.

| Role | Light value | Dark value | Tailwind class | Usage |
|------|-------------|------------|----------------|-------|
| Dominant (60%) | #f8fafc (slate-50) | #020617 (slate-950) | `bg-slate-50 dark:bg-slate-950` | Page background (SlideLayout) |
| Secondary (30%) | #ffffff (white) | #0f172a (slate-900) | `bg-white dark:bg-slate-900` | Card shells (CARD_CLASSES), nav |
| Accent (10%) | #06b6d4 (cyan-500) | #0891b2 (cyan-600) | `text-cyan-600 dark:text-cyan-400` | See accent reserved list |
| Body text (primary) | #0f172a (slate-900) | #ffffff (white) | `text-slate-900 dark:text-white` | Headings, card titles |
| Body text (secondary) | #475569 (slate-600) | #94a3b8 (slate-400) | `text-slate-600 dark:text-slate-400` | Captions, body text, descriptions |
| Card border | #e2e8f0 (slate-200) | #1e293b (slate-800) | `border-slate-200 dark:border-slate-800` | Card shells, nav border |
| CodeCallout bg | #f1f5f9 (slate-100) | #1e293b (slate-800) | `bg-slate-100 dark:bg-slate-800` | CodeCallout component (do not override) |
| Destructive | N/A | N/A | — | No destructive actions in this phase |

Accent (`cyan`) reserved for:
1. Overline labels only (`text-cyan-600 dark:text-cyan-400` on `text-xs font-semibold tracking-widest uppercase` elements)
2. Slide number badge in SlideLayout nav (managed by SlideLayout — do not touch)
3. Tailwind button (`bg-cyan-500 dark:bg-cyan-600` on `TAILWIND_BTN_CLASSES` — migrated from ButtonComparison.tsx)

Accent is NOT applied to: problem card headings, philosophy intro heading, card body text, naming card content, or any interactive hover states on this slide.

Dark mode mechanism: `@custom-variant dark (&:where(.dark, .dark *))` declared in `globals.css`. Every new element that has a light color must have a corresponding `dark:` class. No `prefers-color-scheme` media queries.

Source: `src/app/globals.css`, `src/components/SlideLayout.tsx`, `src/app/what-is-tailwind/page.tsx` (CARD_CLASSES), `src/components/ButtonComparison.tsx` (TAILWIND_BTN_CLASSES), RESEARCH.md §Dark Mode Checklist — all verified.

## Anti-Patterns (enforcement)

| Pattern | Prohibition | Consequence |
|---------|-------------|-------------|
| Dynamic class interpolation | No `` `bg-${color}-500` `` — Tailwind cannot scan dynamically constructed strings | Classes get purged from production build |
| `"use client"` on page.tsx | Forbidden per CLAUDE.md | Breaks RSC tree; unnecessary re-renders |
| `"use client"` on UtilityButton.tsx | Not needed — no state, no browser APIs | Unnecessary client bundle size |
| Naming card in right column | D-13 locks it to left column | Breaks "semantic CSS = you invent names" teaching narrative |
| `useState` toggle in SemanticButton | D-08 removes click-to-reveal | Defeats always-visible teaching beat |
| `prefers-color-scheme` media query | Not a valid dark mode mechanism here | Conflicts with ThemeToggle class-based approach |
| Duplicated class strings | All class strings must use single-source const | CodeCallout drift; two things to update on each change |
