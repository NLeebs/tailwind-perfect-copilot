
The semantic `.btn` button in the left column is the primary focal point — it is the only interactive element and the first teaching moment.

## Interaction Contract

### Semantic Button CSS Reveal (D-04, D-05, D-10)

| State | Visual |
|-------|--------|
| Default (closed) | Semantic `.btn` button visible. No CSS shown. Implicit affordance that clicking reveals something — no label needed (audience discovers it). |
| Revealed (open) | `showCss === true`: CodeCallout chip appears below the `.btn` button with the full `.btn` CSS definition. Chip uses the exact same `CodeCallout` component as all other callouts. |

Toggle trigger: clicking the semantic `.btn` button itself (not a separate "reveal" button).

Toggle state lives in `ButtonComparison` client island. Transition: `showCss` boolean toggles a conditional render. No animation required for Phase 2 (animations are Phase 7 scope — reduced-motion audit).

### Tailwind Button (D-06)

CodeCallout below the Tailwind button is always rendered — no toggle. The contrast between "click to see" vs. "always there" is the teaching moment.

### No Hover States (D-08)

Neither button has hover styles. `cursor-default` is acceptable; `cursor-pointer` on the semantic button is fine since it's the click trigger for the reveal.

## Single-Source Constants (D-09, D-14)

These constants must be defined once and passed to both the element and its `CodeCallout`:

```
// In ButtonComparison.tsx
const TAILWIND_BTN_CLASSES = "bg-cyan-500 dark:bg-cyan-600 text-white font-semibold px-5 py-3 rounded-lg 3xl:px-8 3xl:py-4";

// In page.tsx (RSC inline card) or a separate RSC component
const CARD_CLASSES = "bg-white dark:bg-slate-900 border border-slate-200 dark:border-slate-800 rounded-2xl p-6 shadow-sm 3xl:p-10";

// In ButtonComparison.tsx (the .btn CSS definition string for reveal)
const BTN_CSS_DEFINITION = `.btn {\n  background-color: #06b6d4;\n  color: white;\n  font-weight: 600;\n  padding: 0.75rem 1.25rem;\n  border-radius: 0.5rem;\n}`;
```

The `.btn` style block on the page (or inside the island) must produce the same visual output as `TAILWIND_BTN_CLASSES`. The `.btn` CSS definition string displayed in the CodeCallout is `BTN_CSS_DEFINITION`.

Source: `02-CONTEXT.md` D-09, D-14, §Established Patterns (single-source callout convention).
