## Spacing Scale

Standard 8-point scale. All token values are multiples of 4. These are the project-wide established values; Phase 5 introduces no new spacing tokens.

| Token | Value | Usage in Phase 5 |
|-------|-------|------------------|
| xs | 4px (`p-1`) | Inline padding on CodeCallout chips |
| sm | 8px (`p-2`, `gap-2`) | Overline-to-callout gap (`mb-2`), tight stacks |
| md | 16px (`p-4`, `gap-4`) | Default element spacing |
| lg | 24px (`p-6`, `gap-6`) | Two-column grid gap (`gap-6`); `3xl:gap-12` at 1920px |
| xl | 32px (`mt-8`) | Section-internal top margin (`mt-8`; `3xl:mt-12`) |
| 2xl | 48px | — |
| 3xl | 64px (`mt-16`) | Spacer `<div>` between three stacked sections (`mt-16`; `3xl:mt-24`) |

Exceptions:
- Section spacer divs use `mt-16 3xl:mt-24` (64px → 96px) — already established in Phases 3 and 4. Source: CONTEXT.md D-04.
- Brand swatch demo block height: `h-16 3xl:h-24` (64px → 96px) — decorative; within the 8-point scale.

## Dark Mode Contract

Dark mode is governed by `@custom-variant dark (&:where(.dark, .dark *))` in `globals.css`. The `ThemeToggle` component adds/removes the `dark` class on `<html>`. No `prefers-color-scheme` media query is used.

| Element | Light | Dark |
|---------|-------|------|
| Page background | `white` / `slate-50` | `slate-900` / `slate-950` (via SlideLayout) |
| Overline labels | `text-cyan-600` | `dark:text-cyan-400` |
| Caption text | `text-slate-600` | `dark:text-slate-400` |
| CodeCallout chip | `bg-slate-100 text-slate-700 border-slate-200` | `dark:bg-slate-800 dark:text-slate-300 dark:border-slate-700` |
| ShikiBlock background | github-light (white/off-white) | github-dark (via `.dark .shiki` CSS override with `!important`) |
| ShikiBlock text | inline `--shiki-light` colors | `.dark .shiki span { color: var(--shiki-dark) !important }` |
| Brand swatch | `bg-brand-500` (#3b82f6) | `bg-brand-500` (#3b82f6) — same value; brand token has no dark variant |
| h1 demo gradient | `from-slate-900 to-slate-500` | `dark:from-white dark:to-slate-400` (global `@layer base` rule) |

Shiki dark mode CSS selector (injected via `<style>` inside `ShikiBlock`):
```css
.dark .shiki,
.dark .shiki span {
  color: var(--shiki-dark) !important;
  background-color: var(--shiki-dark-bg) !important;
}
```
Note: `.dark` (applied to `<html>`) is functionally equivalent to `html.dark` for this project. The `@custom-variant dark` generates `:where(.dark, .dark *)` which includes any element inside `<html class="dark">`. Source: RESEARCH.md §Summary.

## Anti-Patterns (Phase 5 Specific)

These must not appear in any executor implementation:

1. Dynamic class string interpolation — no `` `bg-${color}-500` ``; use `bg-brand-500` directly.
2. `"use client"` on `page.tsx` — all three sections are RSC; `ShikiBlock` is `async` RSC.
3. Adding `--color-brand-500` to `@theme inline {}` — use the plain `@theme {}` block.
4. Reading `globals.css` from disk at runtime (`fs.readFile`) — snippet strings are hardcoded constants.
5. `prefers-color-scheme` in the Shiki override `<style>` tag — use `.dark .shiki` selector only.
6. Missing `!important` on Shiki dark overrides — Shiki embeds inline `style` attributes that beat class selectors without `!important`.
7. Leaving `"05"` in the `stubCases` filter in `slide-pages.test.tsx` after the stub is replaced.
