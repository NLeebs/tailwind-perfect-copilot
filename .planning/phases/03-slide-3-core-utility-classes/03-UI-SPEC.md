## Spacing Scale

Declared values (Tailwind spacing units, multiples of 4):

| Token | Tailwind Class | px Equiv | Usage |
|-------|---------------|---------|-------|
| xs | `gap-1` / `p-1` | 4px | Step node inner gaps, icon-to-text spacing |
| sm | `gap-2` / `p-2` / `mt-2` | 8px | Label spacing below step nodes, callout chip internal gaps |
| md | `gap-4` / `p-4` / `mt-4` | 16px | Default element spacing within card, gap between flex/grid children |
| lg | `gap-6` / `p-6` / `mt-6` | 24px | Card internal padding, column gap in two-column grid |
| xl | `gap-8` / `mt-8` | 32px | Step navigator bottom margin, section gap |
| 2xl | `py-12` / `mt-12` | 48px | Vertical separation between card-builder and flex/grid sections |
| 3xl | `py-20` | 80px | (inherited from SlideLayout `main` padding — do not override) |

**3xl: escalations (1920px breakpoint):**
- Column gap: `gap-6 3xl:gap-12`
- Card padding: `p-6 3xl:p-10`
- Step navigator node size: `size-10 3xl:size-16`
- Section spacing: `mt-12 3xl:mt-20`
- Flex/grid child box size: `size-16 3xl:size-24`
- Navigator row gap: `gap-3 3xl:gap-5`

Exceptions: Step navigator nodes use `size-10` (40px) at default, `size-16` (64px) at 3xl — intentionally larger than the 8-point scale to hit a comfortable click target.

## Component Contracts

### CardBuilder Client Island (`src/components/CardBuilder.tsx`)

**State:** `const [activeStep, setActiveStep] = useState(0)` — zero-indexed (0 = Layout, step 1 in UI).

**STEPS array shape:**
```
const STEPS: StepConfig[] = [
  { label: "Layout",  newClasses: "...", allClasses: "..." },
  { label: "Spacing", newClasses: "...", allClasses: "..." },
  { label: "Typo",    newClasses: "...", allClasses: "..." },
  { label: "Color",   newClasses: "...", allClasses: "..." },
  { label: "Borders", newClasses: "...", allClasses: "..." },
  { label: "Flex",    newClasses: "...", allClasses: "..." },
]
```

**Cumulative card class progression (Claude's discretion — D-09):**

| Step | Label | allClasses (cumulative, applied to card element) | newClasses (shown in CodeCallout) |
|------|-------|--------------------------------------------------|----------------------------------|
| 0 | Layout | `"w-full max-w-sm"` | `"w-full max-w-sm"` |
| 1 | Spacing | `"w-full max-w-sm p-6"` | `"p-6"` |
| 2 | Typo | `"w-full max-w-sm p-6 text-sm font-semibold"` | `"text-sm font-semibold"` |
| 3 | Color | `"w-full max-w-sm p-6 text-sm font-semibold bg-white dark:bg-slate-900 text-slate-900 dark:text-white"` | `"bg-white dark:bg-slate-900 text-slate-900 dark:text-white"` |
| 4 | Borders | `"w-full max-w-sm p-6 text-sm font-semibold bg-white dark:bg-slate-900 text-slate-900 dark:text-white border border-slate-200 dark:border-slate-800 rounded-2xl shadow-sm"` | `"border border-slate-200 dark:border-slate-800 rounded-2xl shadow-sm"` |
| 5 | Flex | `"w-full max-w-sm p-6 text-sm font-semibold bg-white dark:bg-slate-900 text-slate-900 dark:text-white border border-slate-200 dark:border-slate-800 rounded-2xl shadow-sm flex flex-col gap-3"` | `"flex flex-col gap-3"` |

**Card content (Claude's discretion):**
- Heading text: `"Tailwind Card"`
- Body text: `"Utilities compose one step at a time."`

### FlexGridComparison RSC (`src/components/FlexGridComparison.tsx` or inline in page)

**Layout:** Two-column grid matching Phase 2 pattern.
- Outer wrapper: `"grid grid-cols-2 gap-6 3xl:gap-12"`
- Left column: FLEX container + overline + CodeCallout below
- Right column: GRID container + overline + CodeCallout below

**Overline labels:** `"FLEX"` and `"GRID"` — `text-xs font-semibold tracking-widest uppercase text-cyan-600 dark:text-cyan-400 3xl:text-base`

**Flex container class string (single-source const):**
```
const FLEX_CLASSES = "flex flex-row gap-4"
```
Applied to the container element AND passed to `<CodeCallout classes={FLEX_CLASSES} />`.

**Grid container class string (single-source const):**
```
const GRID_CLASSES = "grid grid-cols-3 gap-4"
```
Applied to the container element AND passed to `<CodeCallout classes={GRID_CLASSES} />`.

**Container wrapper:** `"mt-3 p-4 bg-white dark:bg-slate-900 border border-slate-200 dark:border-slate-800 rounded-xl 3xl:mt-4 3xl:p-6"`

**Numbered boxes:** Children 1, 2, 3 — identical markup in both containers.
```
const BOXES = [
  { num: "1", className: "bg-violet-200 dark:bg-violet-800 text-violet-900 dark:text-violet-100 ..." },
  { num: "2", className: "bg-sky-200 dark:bg-sky-800 text-sky-900 dark:text-sky-100 ..." },
  { num: "3", className: "bg-emerald-200 dark:bg-emerald-800 text-emerald-900 dark:text-emerald-100 ..." },
]
```
Box base classes: `"flex items-center justify-center size-16 3xl:size-24 rounded-xl text-lg font-bold 3xl:text-2xl"` (appended to each color class string above).

**CodeCallout placement:** Below each container, `"mt-3 3xl:mt-4"`.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Click step node N | `setActiveStep(N - 1)` — jumps directly to that step |
| Click "Next ►" at step 1–5 | `setActiveStep(prev => prev + 1)` |
| Click "◄ Prev" at step 2–6 | `setActiveStep(prev => prev - 1)` |
| Click "◄ Prev" at step 1 | No-op — button is disabled (`opacity-40 cursor-not-allowed pointer-events-none`) |
| Click "Next ►" at step 6 | No-op — button is disabled (`opacity-40 cursor-not-allowed pointer-events-none`) |
| Initial render | Opens at step 1 (Layout) — `useState(0)` |
| Card classes | Always `STEPS[activeStep].allClasses` — cumulative |
| CodeCallout content | Always `STEPS[activeStep].newClasses` — newly added only |
| Dark mode toggle | All dark: variants respond via `@custom-variant dark` on `<html>` — no component-level logic |
| FlexGridComparison | Static RSC — no interaction |

## Dark Mode Contract

All components use `@custom-variant dark (&:where(.dark, .dark *))` exclusively — no `prefers-color-scheme` media queries anywhere.

Every interactive element and surface must have explicit `dark:` variants:
- Surfaces: `bg-white dark:bg-slate-900` / `bg-slate-50 dark:bg-slate-950`
- Borders: `border-slate-200 dark:border-slate-800`
- Text: `text-slate-900 dark:text-white` / `text-slate-600 dark:text-slate-400`
- Step nodes inactive: `bg-slate-200 dark:bg-slate-700`
- CodeCallout: `bg-slate-100 dark:bg-slate-800 text-slate-700 dark:text-slate-300 border-slate-200 dark:border-slate-700` (locked in CodeCallout.tsx)
- Numbered boxes: each has explicit `dark:bg-*` and `dark:text-*` variants
