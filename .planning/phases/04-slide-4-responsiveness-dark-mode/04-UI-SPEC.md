## Color

### Light / Dark Palette (60/30/10 split)

| Role | Light classes | Dark classes | Usage |
|------|--------------|--------------|-------|
| Dominant surface (60%) | `bg-slate-50` or transparent | `bg-slate-950` or transparent | Page background (provided by SlideLayout) |
| Secondary surface (30%) | `bg-white` | `dark:bg-slate-900` | Profile cards, info card body zone |
| Secondary surface alt | `bg-slate-100` | `dark:bg-slate-800` | Tab buttons (inactive), info card header zone |
| Border | `border-slate-200` | `dark:border-slate-800` | Card borders, tab button borders, CodeCallout border |
| Accent (10%) | `bg-cyan-500 text-white` | same | Active tab button; overline text (cyan-600 / dark:cyan-400) |

Accent reserved for:
1. Active tab button background (`bg-cyan-500 text-white ring-2 ring-cyan-500`)
2. Section overline labels (`text-cyan-600 dark:text-cyan-400`)
3. Info card action button hover state (light mode only: `hover:bg-cyan-600`)

Accent NOT used for: inactive tab buttons, profile card backgrounds, info card body/header surfaces, borders, text body.

### Semantic Colors

| Role | Light | Dark | Reserved for |
|------|-------|------|-------------|
| Body text | `text-slate-900` | `dark:text-white` | Headings, card titles, tab labels |
| Muted text | `text-slate-600` | `dark:text-slate-400` | Description paragraphs, profile card role |
| Subtle text | `text-slate-500` | `dark:text-slate-400` | Profile card role, inactive tab labels |
| Destructive | not applicable | not applicable | No destructive actions in this phase |

### Dark Mode Contract (S4-02 — 6+ dark: classes on info card)

## Component Interaction Contract

### 1. ResponsiveDemo (Client Island — src/components/ResponsiveDemo.tsx)

**State:** `activeTab: 'mobile' | 'tablet' | 'desktop'` via `useState`. Default: `'mobile'`.

**Tab button row:**
```
Layout: flex items-center justify-center gap-3 mb-8 3xl:mb-12
Button (active):   bg-cyan-500 text-white ring-2 ring-cyan-500 ring-offset-2
                   dark:ring-offset-slate-950 font-semibold text-sm 3xl:text-lg
                   px-5 py-2 3xl:px-8 3xl:py-3 rounded-lg cursor-default
Button (inactive): bg-slate-100 dark:bg-slate-800 text-slate-600 dark:text-slate-400
                   font-semibold text-sm 3xl:text-lg px-5 py-2 3xl:px-8 3xl:py-3
                   rounded-lg hover:bg-slate-200 dark:hover:bg-slate-700 transition-colors
```

Tab labels: "Mobile", "Tablet", "Desktop" (no breakpoint notation in label — kept simple for TV legibility).

**Layout class lookup map (single-source, drives both container and CodeCallout):**

| Tab | Container class | CodeCallout string |
|-----|----------------|-------------------|
| mobile | `flex flex-col gap-4` | `flex flex-col gap-4` |
| tablet | `flex flex-row gap-4` | `flex flex-row gap-4` |
| desktop | `grid grid-cols-3 gap-4` | `grid grid-cols-3 gap-4` |

Container wrapper: `mt-6 3xl:mt-8 p-6 3xl:p-8 bg-slate-100 dark:bg-slate-800 rounded-2xl`

**Profile cards (3 identical cards, layout changes per tab):**
- Avatar: `size-12 3xl:size-16 rounded-full bg-slate-300 dark:bg-slate-600` (placeholder circle)
- Name: `text-sm font-semibold text-slate-900 dark:text-white 3xl:text-base`
- Role: `text-xs text-slate-500 dark:text-slate-400 3xl:text-sm`
- Card wrapper: `bg-white dark:bg-slate-900 border border-slate-200 dark:border-slate-800 rounded-xl p-4 3xl:p-6 flex flex-col items-center gap-2 3xl:gap-3`

Sample profile content:
- "Alex Rivera" / "Frontend Engineer"
- "Sam Chen" / "Design Systems Lead"
- "Jordan Park" / "Product Designer"

**CodeCallout strip (below profile container):**
- Layout: full-width strip below the profile container
- Overline above: `text-xs font-semibold tracking-widest uppercase text-cyan-600 dark:text-cyan-400 3xl:text-base mb-2`
- Overline text: "LAYOUT CLASSES"
- CodeCallout: `<CodeCallout classes={LAYOUT_CLASSES[activeTab]} />`

**Single-source const pattern:** `const LAYOUT_CLASSES: Record<Tab, string> = { ... }` — same map entry drives both the container className and the CodeCallout prop.

### 2. DarkModeCard (RSC — inline in page.tsx or src/components/DarkModeCard.tsx)

Pure RSC — no `useState`. ThemeToggle on `<html>` drives all dark: variants automatically.

**Card structure (top-to-bottom zones):**

```
Outer wrapper:  border border-slate-200 dark:border-slate-700 rounded-2xl
                overflow-hidden shadow-sm

Header zone:    bg-slate-100 dark:bg-slate-800 px-8 py-5 3xl:px-12 3xl:py-7
  - Title:      text-lg font-semibold text-slate-900 dark:text-white 3xl:text-2xl
  - Badge:      bg-slate-200 text-slate-700 dark:bg-slate-700 dark:text-slate-200
                text-xs font-semibold px-2 py-0.5 rounded-full 3xl:text-sm

Body zone:      bg-white dark:bg-slate-900 px-8 py-6 3xl:px-12 3xl:py-8
  - Body text:  text-sm text-slate-600 dark:text-slate-400 3xl:text-lg

Action zone:    bg-white dark:bg-slate-900 px-8 pb-6 3xl:px-12 3xl:pb-8
  - Button:     bg-cyan-500 text-white dark:bg-cyan-600 dark:text-white
                hover:bg-cyan-600 dark:hover:bg-cyan-700
                dark:md:hover:bg-sky-600 dark:md:hover:text-white
                font-semibold text-sm 3xl:text-base px-5 py-2 3xl:px-7 3xl:py-3
                rounded-lg transition-colors
```

Sample card content:
- Header title: "Your Dashboard"
- Badge: "Pro"
- Body: "Here's a summary of your recent activity. Toggle dark mode to see how every utility class responds."
- Button label: "View Report"

### 3. CodeCallout Strip (below DarkModeCard, RSC)

**Layout:** stacked vertically (not side-by-side). Two chips, two overlines.

**Chip 1 — All dark: pairs:**
```
Overline: "DARK MODE CLASSES" — text-xs font-semibold tracking-widest uppercase
          text-cyan-600 dark:text-cyan-400 3xl:text-base mb-2 mt-6 3xl:mt-8
CodeCallout: multi-line string (whitespace-pre-wrap renders each pair on its own line)
```

Dark: class pairs string (single-source const `DARK_CLASSES`):
```
dark:bg-slate-800   (header zone)
dark:bg-slate-900   (body zone)
dark:text-white     (title)
dark:text-slate-400 (body text)
dark:border-slate-700
dark:bg-slate-700 dark:text-slate-200   (badge)
dark:bg-cyan-600    (button)
```

**Chip 2 — Stacked variants:**
```
Overline: "STACKED VARIANTS" — text-xs font-semibold tracking-widest uppercase
          text-cyan-600 dark:text-cyan-400 3xl:text-base mb-2 mt-4 3xl:mt-6
CodeCallout: single-line string (single-source const `STACKED_CLASSES`)
```

Stacked variants string (single-source const `STACKED_CLASSES`):
```
dark:md:hover:bg-sky-600 dark:md:hover:text-white
```
