## Spacing Scale

| Token | Tailwind Class | px | Usage |
|-------|----------------|----|-------|
| xs | `gap-1` / `px-1` | 4px | Icon-to-text inline gap |

| lg | `p-6` → `3xl:p-8` | 24–32px | Card padding (all demo panels) |

## Typography

All sizes are base (< 1920px) → 3xl (≥ 1920px). Weights are font-normal (400) and font-semibold (600) only — no other weights used in this project.

| Role | Base Size | 3xl Size | Weight | Line Height | Usage |
|------|-----------|----------|--------|-------------|-------|
| Display / H1 | `text-5xl` (48px) | `3xl:text-7xl` (72px) | bold (700) | 1.1 tight | Slide title — inherited from SlideLayout |
| Overline label | `text-xs` (12px) | `3xl:text-base` (16px) | semibold (600) | 1.0 | Section and panel category labels |
| Body / description | `text-sm` (14px) | `3xl:text-xl` (20px) | normal (400) | 1.5 | Demo card body, helper paragraphs |
| Code / callout | `text-[13px]` | `3xl:text-base` (16px) | normal (400) mono | relaxed (1.625) | CodeCallout component — built-in, do not override |

**Label / button text:** `text-sm 3xl:text-base` at weight semibold (600). Used for panel sub-labels, button text, form labels, and "Toggle Active" / "Toggle" buttons.

Source: `src/components/CodeCallout.tsx` (callout size), `src/components/SlideLayout.tsx` (h1 size), `src/components/CardBuilder.tsx` (body/overline sizes). Source: CLAUDE.md §Architecture.

## Interaction Contract

| Element | Trigger | Behavior | Transition |
|---------|---------|----------|------------|
| Panel 1 card | Mouse hover | `hover:bg-sky-50 dark:hover:bg-slate-700` applies | `transition-colors duration-200` |
| Panel 2 card | Mouse hover on parent `group` | `group-hover:scale-105 group-hover:bg-sky-200` on avatar; `group-hover:text-sky-700` on text | `transition-all duration-200` (avatar), `transition-colors duration-200` (text) |
| Panel 3 toggle button | Click | `isActive` flips; card switches between PANEL3_CLASSES.active and PANEL3_CLASSES.inactive | `transition-colors duration-200` on card |
| Email input | Type invalid text | `peer-invalid:visible` makes sibling `<p>` appear — no JavaScript | CSS `:invalid` pseudo-class, instant |
| "Toggle Active" button | Click | `isActive` flips; `data-active` attribute added/removed; card shifts to/from `bg-purple-600 text-white` | `transition-colors duration-200` on card |

**Hover behavior note:** `hover:` in Tailwind v4 is wrapped in `@media (hover: hover)`. Mouse/trackpad devices trigger hover as expected. Touch devices do not. This is correct for a desktop presentation context.

**No prefers-reduced-motion exceptions required in Phase 6:** All transitions are `duration-200` color/transform shifts of < 300ms — not animations. Phase 7 QA handles the animation audit.

Source: 06-RESEARCH.md §Verified Tailwind v4 Variant Behavior Q4, §Pattern 2 code example.
